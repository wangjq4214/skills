# Leading Words

A leading word is a compact concept that replaces repeated descriptions.

It provides a stable mental anchor for a recurring behavior.

---

## Purpose

Leading words improve predictability by:

- reducing repeated explanations
- creating shared terminology
- improving recognition of important patterns

A good leading word lets the agent recall a larger concept from a small phrase.

---

## When to Create a Leading Word

Create a leading word when:

- the same concept appears repeatedly
- multiple instructions describe the same behavior
- a concept needs consistent recognition across skills
- a long explanation can be replaced by a shorter concept

Do not create leading words for every idea.

The purpose is compression, not terminology expansion.

---

## Good Leading Words

A good leading word is:

- short
- memorable
- concrete
- connected to an existing concept

Examples:

    fast + deterministic + low-overhead execution cycle

becomes:

    tight loop

---

    one authoritative definition

becomes:

    single source of truth

---

    detected failure condition

becomes:

    red state

---

## Bad Leading Words

Avoid:

- vague adjectives
- marketing terms
- arbitrary abbreviations
- concepts that only exist inside one skill

Examples:

Bad:

    efficient-quality-process

Bad:

    advanced-reliability-framework

Better:

    tight loop

---

## Usage Rule

A leading word should replace explanation.

It should not create another layer that requires explanation.

Before creating a new term, ask: "Will this concept appear often enough that a stable name improves execution?"

- If no: keep the original wording.

---

## Consistency Rule

Once a leading word is established:

- use it consistently
- do not introduce synonyms
- do not redefine it elsewhere

A leading word works only when it becomes a stable reference.
